# 教师工作台基础产品需求文档 (PRD)

## 1. 文档概述
| 项目 | 内容 |
| :--- | :--- |
| **文档名称** | 教师工作台（思辨智教）基础产品需求文档 |
| **版本号** | V1.0 |
| **状态** | 草稿 (Demo开发用) |
| **目标用户** | K12 教师 |

## 2. 产品背景与目标
本产品旨在为教师提供一个智能化的教学设计工作台，通过“目标先行”的设计逻辑和“问答链”核心方法论，辅助教师完成从备课到作业设计的全流程。
**核心目标**：开发一个可交互的产品 Demo，展示核心的 AI 备课流程，特别是问答链的生成与落实。

## 3. 产品架构与交互框架
### 3.1 整体布局 (参考交互框架)
采用经典的 **左侧导航 + 中间工作区** 布局，强调 AI 助手的对话与生成能力。

*   **左侧导航栏 (Sidebar)**：
    *   **一级导航**：
        1.  **AI 备课** (展开包含二级菜单)
        2.  **作业设计**
        3.  **AI 评课**
    *   **二级菜单 (AI 备课下)**：
        *   **教学设计**
        *   **素材创生**
    *   **历史任务**：展示历史生成的教学设计记录。
    *   **历史任务**：展示历史生成的教学设计记录 (如“《背影》层进式问答链设计”)。
    *   **用户信息**：底部展示用户头像与设置。

*   **主工作区 (Main Workspace)**：
    *   **欢迎页/对话页**：展示欢迎语 "Hi, 我是您的思辨教学助手"。
    *   **核心输入区**：支持自然语言输入 (如“请输入课程名称或核心知识点”)，支持语音输入和文件上传。
    *   **快捷生成卡片**：提供“问答链生成”、“教案大纲”、“教学目标”等快捷入口。
    *   **生成结果页**：左侧为对话流/大纲，右侧为结构化编辑预览区 (支持富文本编辑与调整)。

## 4. 核心功能模块详解

### 4.1 教学目标设计 (Target Design)
**设计逻辑**：目标先行。所有后续设计 (问题、活动、作业) 均需锚定教学目标。

*   **功能要点**：
    *   **章节/课时目标设定**：用户输入课题后，系统首先引导确定教学目标。
    *   **默认目标调用**：系统自动调用数据库中的标准课程标准/教材目标推荐给教师。
    *   **校本化调整**：支持教师对推荐目标进行修改、增删，形成个性化/校本化目标。
    *   **目标锁定**：目标确认后，作为后续生成的“元数据”被锁定，确保一致性。

### 4.2 问答链设计 (Q&A Chain Design)
**设计逻辑**：基于“四何问题” (Who, What, Why, How) 构建层进式问题链。

*   **功能要点**：
    *   **主问题提炼**：基于教学目标，AI 自动生成 1-3 个核心主问题。
    *   **问题链构建**：
        *   **理科类**：侧重逻辑递进 (如：现象 -> 原理 -> 应用)。
        *   **文科类**：侧重发散思维与多视角探讨 (如：情感 -> 意象 -> 时代背景)。
    *   **支架式设计**：在生成过程中展示“四何”标签，帮助教师理解问题设计的维度。
    *   **编辑与优化**：教师可调整问题顺序、修改措辞，AI 实时检查问题与目标的匹配度。

### 4.3 情境创设与活动设计 (Scenario & Activity)
**设计逻辑**：问答链的落地载体。情境服务于问题解决。

*   **功能要点**：
    *   **情境生成**：根据核心问题，AI 推荐贴合学生生活的真实情境或虚拟情境 (如科学实验场景、历史角色扮演)。
    *   **活动步骤化**：将问答链转化为具体的教学活动步骤 (Step-by-Step)。
    *   **形式匹配**：
        *   简单问题 -> 个人思考/抢答。
        *   复杂/发散问题 -> 小组合作/探究活动。
    *   **可视化呈现**：在教案视图中，清晰展示“目标 -> 问题 -> 情境 -> 活动”的对应关系。

### 4.4 AI作业 (Homework Design)
**设计逻辑**：教学目标的检测与延伸。

*   **功能要点**：
    *   **目标对齐检测**：每一道生成的题目都需标注其对应的教学目标。
    *   **题库资源整合**：调用题库资源，精准匹配练习题。
    *   **问答链扩展**：
        *   **基础题**：检测课堂核心问题的掌握。
        *   **拓展题**：在作业中设置延伸性问题，引导课后思考 (思维训练)。
    *   **分层设计**：自动生成基础、进阶、挑战三个难度的作业单。

### 4.5 AI评课 (Classroom Analysis)
**设计逻辑**：闭环反馈。

*   **功能要点**：
    *   **落实度分析**：(Demo阶段可设计为模拟数据或上传录音后的分析报告) 分析课堂实录中，教师是否有效提出了预设的“问答链”。
    *   **情境有效性**：分析学生在特定情境下的参与度与互动质量。
    *   **改进建议**：基于分析结果，反馈教学设计 (如：某问题过难，学生无响应)，建议调整下一次的教学目标或问题设计。

## 5. Demo 开发优先级 (MVP)
1.  **P0 (最高优先级)**：
    *   工作台整体框架搭建 (左侧导航 + 对话交互区)。
    *   **教学目标设计**：输入课题 -> 推荐目标 -> 修改确认。
    *   **问答链生成**：基于目标 -> 生成层进式问答链 (展示理科/文科逻辑)。
2.  **P1 (次高优先级)**：
    *   **情境与活动生成**：将问答链包装为活动卡片。
    *   **作业生成**：基于目标的简单习题推荐。
3.  **P2 (待定)**：
    *   课堂分析模块 (仅展示静态报表页面即可)。
    *   素材创生工具。
