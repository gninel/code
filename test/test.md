7.2 对话流程示例：教学改进助手
1. 用户进入 → AI欢迎语 + 操作引导
   "您好！我是你的教学改进助手。我可以帮你：
    • 诊断你的教学设计
    • 优化课堂提问
    • 分析学生学情"

2. 用户选择课程
   AI: "请选择要分析的课程"
   → 下拉列表 / 搜索框

3. AI自动分析
   → 显示分析中...（3-5秒）
   → 展示分析结果（带图表）

4. 交互建议
   "我发现以下3个改进机会：
    1. [机会描述] → [查看详情] [应用建议]
    2. [机会描述] → [查看详情] [应用建议]
    3. [机会描述] → [查看详情] [应用建议]"

5. 用户选择应用
   → 一键将建议应用到备课设计
   → 显示修改前后对比
   → 保存或恢复

6. 后续追问
   支持用户提出后续问题，继续优化

十、交互和设计规范

色彩：保持现有蓝色主调，强调AI功能用浅紫色
字体：标题16px/加粗，正文14px/常规，标签12px
间距：统一8px网格系统
圆角：按钮8px，卡片4px
阴影：浅色背景卡片，0 2px 4px rgba(0,0,0,0.08)

10.2 交互规范

加载状态：显示骨架屏或Loading动画
反馈提示：操作成功/失败使用Toast通知
确认对话：危险操作（删除）需二次确认
空状态：无数据时显示相关引导图和建议

10.3 响应式设计

桌面版（≥1200px）：左侧导航 + 3列推荐卡片
